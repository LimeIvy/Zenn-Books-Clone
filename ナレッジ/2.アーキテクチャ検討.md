# 使用技術の選定とその理由

## フロントエンド

- 機能を追加するごとに基本的にパスが増えること（SPAではない）
- MVP開発を行い、時間が余れば追加で機能を開発すること
- ブログアプリであり、記事ごとのSEOも重要

これらの観点から、ページ遷移が多くてSEOも重要なブログ投稿アプリであることから、**Next.js**を採用した。

## バックエンド

- リアルタイム性やパフォーマンスは重視しないが、堅牢性は重視したい
- フロントとバックで型定義を共有して開発したい

これらの理由から、バックエンドでは高速で必要最低限のものがそろったうえで、Next.jsと型をそろえて開発しやすい**Hono**を採用した。

## データベース

- データ更新を頻繁に行わない
- 複雑なクエリが必要なく、簡単なCRUD操作程度
- 将来的には検索機能などを実装

これらの観点から、**MySQL**を採用した。

# フロントとバックエンドの役割分担

## フロント側
- UIを担当

## バック側
- API設計
- CRUD操作を行い、DBに保存するまでを担当

# APIの簡易設計

- GET /books（本一覧の取得）
- POST /books（本の新規追加）
- GET /books/:id（本の詳細情報の取得・全チャプター取得）
- PUT /books/:id（本の詳細情報の更新）
- DELETE /books/:id（本の削除）
  
- GET /chapters (チャプターの詳細情報の取得)
- POST /chapters（チャプターの追加）
- PUT /chapters/:id（チャプターの更新）
- DELETE /chapters/:id（チャプターの削除）
